# Boorapatis Ploysuwan — เว็บไซต์ส่วนตัว / เว็บไซต์ผู้สร้างคอนเทนต์

เว็บไซต์ Static HTML/CSS/JS (Vanilla JS, ไม่มี framework, ไม่มีระบบหลังบ้าน) สำหรับรวบรวมและนำเสนอ
ช่อง YouTube และ TikTok ของ Boorapatis Ploysuwan เพื่อเผยแพร่ความรู้ แนวคิด และสื่อสร้างสรรค์
ที่เป็นประโยชน์ต่อสังคมไทย

---

## 1. รายการไฟล์ในโปรเจกต์

```
.
├── index.html              หน้าเว็บไซต์หลัก (ทุก section)
├── privacy.html             นโยบายความเป็นส่วนตัว
├── terms.html               ข้อตกลงการใช้งาน
├── content-policy.html      นโยบายเนื้อหา (ละเอียดกว่าหัวข้อ "หลักการเนื้อหา" ในหน้าแรก)
├── styles.css               สไตล์ทั้งหมดของเว็บไซต์ (ใช้ร่วมกันทุกหน้า)
├── script.js                 การทำงานแบบ interactive (เมนูมือถือ, scroll reveal, ปุ่มกลับขึ้นบน ฯลฯ — ใช้ร่วมกันทุกหน้า)
├── robots.txt                กำหนดการเข้าถึงของ search engine
├── sitemap.xml                แผนผัง URL สำหรับ SEO (รวม 4 หน้าทั้งหมด)
├── README.md                 ไฟล์นี้
└── assets/
    └── favicon.svg            ไอคอนเว็บไซต์แบบ placeholder (SVG รูปคลื่นสัญญาณ)
```

## 2. วิธีแก้ไขข้อมูลส่วนตัว

ข้อมูลทั้งหมดอยู่ใน `index.html` สามารถแก้ไขได้โดยค้นหาส่วนต่อไปนี้:

| ต้องการแก้ไข | อยู่ตรงไหนใน index.html |
|---|---|
| ชื่อ / โลโก้ | แท็ก `<span class="logo-text">` ในส่วน Header |
| ข้อความ Hero | ส่วน `<section class="hero" id="top">` |
| เนื้อหา About | ส่วน `<section class="section about" id="about">` |
| ลิงก์ช่อง YouTube/TikTok | ส่วน `<section class="section channels" id="channels">` (แท็ก `<a href="...">` ในแต่ละ `.channel-card`) |
| อีเมลติดต่อ | ค้นหาคำว่า `burapatis@gmail.com` (มีอยู่ 2 ตำแหน่ง: ส่วน Contact และ `mailto:`) |
| ปีที่แสดงใน footer | อัปเดตอัตโนมัติด้วย JavaScript (`script.js`) ไม่ต้องแก้ไขมือ |

## 3. วิธีเปลี่ยนโดเมนใน sitemap.xml และ meta tags

ไฟล์เหล่านี้ใช้โดเมน placeholder คือ `https://example.com/` กรุณาแก้ไขให้เป็นโดเมนจริงของคุณ
**ก่อน** นำเว็บไซต์ขึ้นใช้งานจริง โดยค้นหาคำว่า `example.com` ในไฟล์ต่อไปนี้แล้วแทนที่ทั้งหมด:

1. `index.html`, `privacy.html`, `terms.html`, `content-policy.html` — แท็ก `<link rel="canonical">`,
   `<meta property="og:url">` ในทุกหน้า และใน `index.html` เพิ่มเติมที่ `<meta property="og:image">`,
   `<meta name="twitter:image">` และ JSON-LD (`@graph` ส่วน Person/WebSite)
2. `robots.txt` — บรรทัด `Sitemap: https://example.com/sitemap.xml`
3. `sitemap.xml` — แท็ก `<loc>` ทั้ง 4 รายการ (หน้าแรก + privacy + terms + content-policy)

ทิป: ใช้ฟังก์ชัน "ค้นหาและแทนที่ทั้งหมด" (Find & Replace All) ในโปรแกรมแก้ไขข้อความ
แล้วแทนที่ `example.com` ด้วยโดเมนจริงของคุณ เช่น `boorapatis.com`

## 4. วิธีเปิดใช้งานแบบ Local (ทดสอบในเครื่องตัวเอง)

ไม่จำเป็นต้องติดตั้งอะไรเพิ่ม เพราะเป็น static website ล้วน ๆ:

**วิธีที่ 1 — เปิดไฟล์ตรง ๆ**
ดับเบิลคลิกที่ `index.html` เพื่อเปิดในเบราว์เซอร์ได้ทันที

**วิธีที่ 2 — รัน local server (แนะนำ เพื่อให้ฟอนต์/บางฟีเจอร์ทำงานสมบูรณ์ที่สุด)**
```bash
# ใช้ Python (มีในเครื่องส่วนใหญ่อยู่แล้ว)
python3 -m http.server 8000
# จากนั้นเปิดเบราว์เซอร์ไปที่ http://localhost:8000
```

## 5. วิธีนำขึ้น GitHub Pages (แบบย่อ)

1. สร้าง repository ใหม่บน GitHub (เช่น `boorapatis-website`)
2. อัปโหลดไฟล์ทั้งหมดในโปรเจกต์นี้ขึ้น repository (push ผ่าน `git` หรือลากไฟล์ผ่านหน้าเว็บ GitHub)
3. ไปที่ **Settings → Pages**
4. เลือก Source เป็น branch `main` (หรือ branch ที่ใช้งาน) และโฟลเดอร์ `/ (root)`
5. กด Save แล้วรอสักครู่ GitHub จะให้ URL เว็บไซต์ เช่น `https://username.github.io/boorapatis-website/`
6. หลังได้โดเมนจริงแล้ว กลับไปทำตามหัวข้อ 3 เพื่ออัปเดต meta tags / sitemap / robots.txt

*(วิธีเดียวกันนี้สามารถใช้ deploy บน Netlify ได้เช่นกัน โดยลากโฟลเดอร์ทั้งหมดไปวางในหน้า Netlify Drop)*

## 6. ข้อควรระวังก่อนเผยแพร่จริง

- **ลิขสิทธิ์รูปภาพ:** เว็บไซต์นี้ไม่มีการใช้รูปภาพจากภายนอกเลย (ใช้ gradient/SVG/emoji แทนทั้งหมด)
  เพื่อหลีกเลี่ยงปัญหาลิขสิทธิ์ หากต้องการเพิ่มรูปภาพจริง (เช่น รูปโปรไฟล์ หรือภาพปกช่อง)
  ให้ใช้ภาพที่คุณเป็นเจ้าของสิทธิ์เองเท่านั้น หรือภาพที่มีสิทธิ์ใช้งานชัดเจน (เช่น ซื้อ license
  หรือเป็นภาพที่อนุญาตให้ใช้งานเชิงพาณิชย์ได้)
- **ตรวจสอบลิงก์ช่อง:** ตรวจสอบ URL ของช่อง YouTube/TikTok ทุกครั้งก่อนเผยแพร่ ว่ายังถูกต้องและใช้งานได้
- **ตรวจทานเนื้อหาก่อนเผยแพร่:** ทุกข้อความในเว็บไซต์ควรได้รับการตรวจทานโดยเจ้าของเว็บไซต์เอง
  ก่อนนำขึ้นใช้งานจริง เพื่อให้แน่ใจว่าถูกต้อง เหมาะสม และเป็นตัวแทนของผู้จัดทำอย่างแท้จริง
- **Favicon:** ไฟล์ `assets/favicon.svg` เป็น placeholder รูปคลื่นสัญญาณสีทอง-น้ำเงิน
  แนะนำให้แทนที่ด้วยโลโก้จริงเมื่อมี
- **ข้อความ disclaimer และนโยบายเนื้อหา:** มีอยู่แล้วทั้งแบบสรุปในหน้าแรก (ส่วน "ข้อจำกัดความรับผิดชอบ")
  และแบบละเอียดในหน้า `privacy.html`, `terms.html`, `content-policy.html` อย่างไรก็ตาม เอกสารเหล่านี้
  จัดทำขึ้นเพื่อให้ข้อมูลทั่วไป **ไม่ถือเป็นคำแนะนำทางกฎหมาย** หากต้องการความครอบคลุมทางกฎหมายที่แม่นยำ
  ตามกรณีใช้งานจริงของคุณ (เช่น การปฏิบัติตาม PDPA อย่างเต็มรูปแบบ) แนะนำให้ปรึกษาผู้เชี่ยวชาญด้านกฎหมาย
  เพื่อตรวจสอบและปรับปรุงเนื้อหาให้เหมาะสมก่อนเผยแพร่จริง

## 8. เทคโนโลยีที่ใช้

- HTML5 (semantic markup)
- CSS3 (custom properties / CSS variables, responsive grid, ไม่ใช้ framework)
- JavaScript แบบ Vanilla (ไม่มี dependency ภายนอก)
- Google Fonts: Noto Sans Thai (เนื้อหา) และ Noto Serif Thai (หัวข้อ) — ใช้งานได้ฟรีตามสิทธิ์ของ Google Fonts

ไม่มีการเรียกใช้ระบบหลังบ้าน ฐานข้อมูล หรือ API ภายนอกใด ๆ เว็บไซต์นี้จึงสามารถนำไปอัปโหลดบน
hosting แบบ static ได้ทุกแห่ง เช่น GitHub Pages, Netlify, Vercel, Cloudflare Pages เป็นต้น
